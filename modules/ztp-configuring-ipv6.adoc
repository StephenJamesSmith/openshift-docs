= Configuring IPv6 addresses for a disconnected environment

Optionally, you can configure IPV6 addresses for the managed clusters.

.Procedure

. Modify the IP addresses in the AgentClusterInstall custom resource for IPv6:
+
[source,yaml]
----
apiVersion: extensions.hive.openshift.io/v1beta1
kind: AgentClusterInstall
metadata:
  # Only include the annotation if using OVN, otherwise omit the annotation
  annotations:
    agent-install.openshift.io/install-config-overrides: '{"networking":{"networkType":"OVNKubernetes"}}'
  name: <cluster-name>
  namespace: <cluster-name>
spec:
  clusterDeploymentRef:
    name: <cluster-name>
  imageSetRef:
    name: <cluster-image-set>
  networking:
    clusterNetwork:
    - cidr: "fd01::/48"
      hostPrefix: 64
    machineNetwork:
    - cidr: <machine-network-cidr>
    serviceNetwork:
    - "fd02::/112"
  provisionRequirements:
    controlPlaneAgents: 1
    workerAgents: 0
  sshPublicKey: <public-key>
----

. Use the IPv6 addresses you defined in any place that requires an IP address.
